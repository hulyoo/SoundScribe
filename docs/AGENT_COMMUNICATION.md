# SoundScribe Agent 沟通机制 v2.0

## 当前问题分析

### 1.0 版本问题
- ❌ 邮件形式松散，缺乏结构化
- ❌ 任务进度不透明
- ❌ Agent 间协作流程不清晰
- ❌ 缺少统一的进度追踪

---

## 2.0 改进方案

### 沟通渠道

| 渠道 | 用途 | 频率 |
|------|------|------|
| **周报** | 各 Agent 向 PM 汇报 | 每周五 |
| **任务板** | 任务状态追踪 | 实时更新 |
| **协作邮件** | Agent 间协作讨论 | 需要时 |
| **紧急邮件** | 阻塞问题、重大变更 | 立即 |

### 邮件格式标准化

```
【类型】[功能/UI/测试/协作] 主题
---
发件人: xxx@soundscribe.local
收件人: xxx@soundscribe.local
抄送: pm@soundscribe.local, boss@soundscribe.local
优先级: P0/P1/P2

## 任务进展
-

## 遇到问题
-

## 下周计划
-

## @相关 Agent
-
```

### 任务看板

文件: `docs/taskboard.md`

| 任务ID | 任务名 | 负责Agent | 状态 | 截止日 |
|--------|--------|-----------|------|--------|
| T001 | Flutter项目初始化 | Backend | Done | 03/07 |
| T002 | 麦克风录音功能 | Frontend | In Progress | 03/10 |
| ... | ... | ... | ... | ... |

### 状态定义

| 状态 | 说明 |
|------|------|
| Pending | 待开始 |
| In Progress | 进行中 |
| Blocked | 阻塞（需协助） |
| Review | 待评审 |
| Done | 已完成 |

---

## Agent 协作流程

```
PM (任务分配)
    ↓
各 Agent 领取任务 → 执行
    ↓
遇到问题 → 发起协作邮件 @相关Agent
    ↓
完成 → 更新任务板状态 → 周报汇报
    ↓
PM 汇总 → 周五向 Boss 汇报
```

---

## 紧急情况处理

1. **阻塞问题**: 立即发邮件 @PM，PM 协调解决
2. **进度延迟**: 提前1天通知 PM，更新任务板
3. **需求变更**: PM 评估影响，通知相关 Agent
